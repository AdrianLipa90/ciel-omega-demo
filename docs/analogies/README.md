# Analogie — warstwa edukacyjna i mnemoniczna

Ten folder zbiera **analogie robocze** dla architektury CIEL/Ω i Omegi.

Cel tego layera nie jest ornamentalny.
Analogie mają tu pełnić trzy funkcje naraz:

1. **dydaktyczną** — pomagają tłumaczyć trudne pojęcia prostym językiem,
2. **mnemoniczną** — pomagają zapamiętywać zależności między pojęciami,
3. **operacyjną** — tworzą most między formalizmem a interfejsem, dokumentacją i nauką.

---

## Zasada użycia

Analogia:
- **nie zastępuje definicji formalnej**,
- **nie unieważnia matematyki**,
- **nie tworzy ontologii sama z siebie**,
- ma być traktowana jako **wierne przybliżenie intuicyjne**.

Kolejność jest zawsze taka:

**formalizm -> interpretacja -> analogia -> pamięć**

Nie odwrotnie.

---

## Zawartość folderu

- `ANALOGY_REGISTRY.md` — rejestr analogii i mapowanie pojęcie -> obraz mnemoniczy
- `MNEMONIC_BOOK_FOR_KIDS.md` — księga mnemoniczna dla dzieci / początkujących
- `TRUTH_ATTRACTOR_ANALOGIES.md` — analogie dla prawdy jako atraktora, więzów, nodes i zbieżności

## Zakres pojęć

Ta warstwa ma tłumaczyć m.in.:

- prawdę jako atraktor,
